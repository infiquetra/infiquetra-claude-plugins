# API Reference Template

Standard structure for SDK API reference documentation.

## Class/Module Organization

### Client Class
- Constructor and initialization
- Configuration options
- Public methods
- Properties

### Model Classes
- Data structures
- Validation rules
- Serialization

### Exception Classes
- Error hierarchy
- Error codes
- Recovery strategies

## Method Documentation Format

```
### method_name(param1, param2)

**Description**: What the method does.

**Parameters**:
- `param1` (type): Description
- `param2` (type, optional): Description (default: value)

**Returns**: Return type and description

**Raises**:
- `ExceptionType`: When this exception occurs

**Example**:
```language
code example
```
```

## Best Practices

1. Document all public APIs
2. Include working examples
3. Document exceptions
4. Specify parameter types
5. Show return value structure
6. Link related methods
7. Version compatibility notes
